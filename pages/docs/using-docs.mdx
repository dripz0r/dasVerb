# Using the Documentation System

This guide explains how to use and contribute to DasVerb's documentation system.

## Accessing the Documentation

The documentation is available in two ways:

1. **Local Development**
   ```bash
   npm run dev
   # Visit http://localhost:3000/docs
   ```

2. **Production**
   - Visit `https://dasverb.com/docs` (when deployed)

## Documentation Structure

```
docs/
├── index.mdx              # Main landing page
├── getting-started.mdx    # Setup guide
├── project-structure.mdx  # Codebase organization
├── changelog.mdx         # Version history
├── using-docs.mdx        # This guide
└── blog/                 # Development blog
    ├── index.mdx
    ├── setting-up-nextra.mdx

    └── ui-ux-improvements.mdx
```

## Writing Documentation

### 1. Using MDX

MDX combines Markdown with React components:

```mdx
# My Page Title

Regular markdown content here.

<MyComponent prop="value" />

## Code Examples

```typescript
const example = 'code';
```
```

### 2. Available Components

Nextra provides several built-in components:

```mdx
<Tabs>
  <Tab title="First Tab">
    Content for first tab
  </Tab>
  <Tab title="Second Tab">
    Content for second tab
  </Tab>
</Tabs>

<Callout type="info">
  This is an informational callout
</Callout>

<Steps>
  <Step title="First Step">
    First step content
  </Step>
  <Step title="Second Step">
    Second step content
  </Step>
</Steps>
```

### 3. Frontmatter

Add metadata to your pages:

```mdx
---
title: 'Page Title'
description: 'Page description'
---

Content here...
```

## Styling

### 1. Using Tailwind

The documentation uses Tailwind CSS:

```mdx
<div className="bg-blue-500 text-white p-4 rounded-lg">
  Styled content
</div>
```

### 2. Custom Components

Create reusable components in `components/docs/`:

```typescript
// components/docs/FeatureCard.tsx
export function FeatureCard({ title, description }) {
  return (
    <div className="border rounded-lg p-4">
      <h3>{title}</h3>
      <p>{description}</p>
    </div>
  );
}
```

## Navigation

### 1. Sidebar Configuration

Configure the sidebar in `theme.config.tsx`:

```typescript
export default {
  sidebar: {
    defaultMenuCollapseLevel: 1,
    titleComponent: ({ title }) => (
      <span className="font-bold">{title}</span>
    ),
  },
}
```

### 2. Adding Pages

Add new pages to the navigation:

```typescript
export default {
  sidebar: {
    items: [
      {
        title: 'Getting Started',
        path: '/getting-started',
      },
      {
        title: 'Features',
        items: [
          {
            title: 'AI Helper',
            path: '/features/ai-helper',
          },
        ],
      },
    ],
  },
}
```

## Search

The documentation includes a search feature:

1. Press `Ctrl/Cmd + K` to open search
2. Type to search across all pages
3. Use arrow keys to navigate results

## Contributing

### 1. Adding Content

1. Create a new `.mdx` file in the appropriate directory
2. Add frontmatter
3. Write content using MDX
4. Submit a pull request

### 2. Editing Existing Content

1. Find the file in the `docs/` directory
2. Make your changes
3. Submit a pull request

### 3. Best Practices

- Keep content concise and clear
- Use code examples when helpful
- Include screenshots for UI features
- Update the changelog for significant changes

## Customization

### 1. Theme

Customize the theme in `theme.config.tsx`:

```typescript
export default {
  logo: <span>DasVerb</span>,
  project: {
    link: 'https://github.com/yourusername/dasVerb',
  },
  // More customization options...
}
```

### 2. Styling

Add custom styles in `styles/docs.css`:

```css
/* Custom styles for documentation */
.docs-content {
  max-width: 800px;
  margin: 0 auto;
}
```
