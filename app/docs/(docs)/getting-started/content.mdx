# Getting Started

Welcome to DasVerb! This guide will help you set up the project and start contributing.

## Prerequisites

Before you begin, make sure you have:

- Node.js 18+ installed
- npm or yarn package manager
- Git
- A code editor (VS Code recommended)
- Google Cloud account (for AI features)

## Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/dasVerb.git
   cd dasVerb
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Set up environment variables:
   Create a `.env.local` file in the root directory with:
   ```
   GOOGLE_AI_API_KEY=your_api_key_here
   ```

4. Start the development server:
   ```bash
   npm run dev
   ```

## Project Structure

The project follows a modern Next.js 14 structure:

```
dasVerb/
├── src/
│   ├── app/           # Main Next.js App Router (routes, pages, layouts, API)
│   ├── components/    # All React components (UI, features, layout, shared)
│   ├── ai/            # AI logic, Genkit flows, and integration with Gemini
│   ├── lib/           # Shared utilities and helper functions
│   ├── hooks/         # Custom React hooks
│   └── types/         # TypeScript type definitions (if present)
├── public/            # Static assets (images, icons, etc.)
├── styles/            # Global styles (Tailwind, custom CSS)
├── pages/docs/        # Official documentation (Nextra)
├── ...                # Config files, scripts, etc.
```

For more details, see our [Project Structure](/docs/project-structure) guide.

## Development Workflow

1. **Branch Strategy**
   - `main`: Production-ready code
   - `develop`: Development branch
   - Feature branches: `feature/feature-name`
   - Bug fixes: `fix/bug-name`

2. **Code Style**
   - Use TypeScript
   - Follow ESLint rules
   - Format with Prettier
   - Write meaningful comments

3. **Testing**
   - Write unit tests for utilities
   - Test components with React Testing Library
   - Run tests before committing
